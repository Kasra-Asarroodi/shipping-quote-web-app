# Development History

## Project Goal

The aim of this project was to build a complete shipping quote website for Pack & Send Unley.

Customers can calculate shipping prices, submit enquiries and view their enquiry history, while administrators can manage customer enquiries through a secure dashboard.

---

## Stage 1

The project began as a simple Flask website capable of calculating shipping prices.

At this stage there was no database and no customer login system.

---

## Stage 2

After completing the pricing system, I added the ability for customers to submit enquiries.

Initially, the project stored data using JSON files.

Although this worked, I realised that a proper database would provide a much better long-term solution.

---

## Stage 3

The project was migrated from JSON storage to SQLite.

This was one of the biggest learning experiences of the project because I learned:

- SQL
- Database design
- CRUD operations
- Separating database logic from application logic

---

## Stage 4

The application was then modularised by separating the project into multiple Python files.

This significantly improved readability and made the code much easier to maintain.

---

## Stage 5

The final stage focused on improving the user experience by adding:

- Customer enquiry dashboard
- Administrator dashboard
- Promotional discount management
- Better validation
- Cleaner code
- Improved documentation

---

## Reflection

This project taught me much more than Flask development.

It introduced me to software architecture, databases, deployment, GitHub, version control and writing maintainable code.

Looking back at the first version of this project, I can clearly see how much both the application and my programming skills have improved.
